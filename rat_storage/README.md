# RAT Storage Service

A dedicated Flask-based microservice designed to store, manage, and serve scraped data (HTML sources, screenshots, and PDFs). This service includes automated cleanup utilities and secure access via timed tokens.

## 🚀 Features

* **Secure Uploads**: API-Key restricted file uploads for the scraper.
* **Encapsulated Storage**: Serves specific files (HTML/JPG) directly from inside ZIP archives.
* **Timed Access**: Uses `itsdangerous` URL signatures to prevent unauthorized file access.
* **Maintenance**: Includes a cleanup script to remove "orphaned" files not present in the database.
* **Documentation Ready**: Fully documented with Google-style docstrings for easy HTML generation.

---

## 🛠 Project Structure

* `storage_service.py`: The main Flask application (Storage API).
* `clean_orphans.py`: Maintenance utility to sync physical storage with the database.
* `test_api.py`: End-to-end testing script for validation.
* `requirements.txt`: Python dependencies.
* `setup.sh`: Automated environment setup script.

---

## 📦 Installation & Setup

### 1. Automatic Setup (Linux)
Run the setup script to create the directory structure and virtual environment:
```bash
chmod +x setup.sh
./setup.sh

```

### 2. Configuration

Edit the `API_KEY` and `DB_URI` in the scripts to match your environment:

* **API_KEY**: Used for both header authentication and ticket signing.
* **STORAGE_FOLDER**: Default is `/var/www/rat/storage/sources/`.

### 3. Running the Service (Development)

```bash
source venv/bin/activate
python storage_service.py

```

*The service defaults to port `5001`.*

---

## 🔒 Security Workflow

The service uses a two-layer security approach:

1. **Upload Protection**: Requires a custom header `X-API-Key`.
2. **View Protection**: Requires a `ticket` parameter. This ticket is a signed, timed HMAC token. Even if a filename is known, it cannot be accessed without a valid ticket generated by the main application.

---

## 🧹 Maintenance (Orphan Cleanup)

To keep your storage folder clean, run the cleanup utility periodically. This script compares the files in the storage directory against the records in your PostgreSQL database.

```bash
source venv/bin/activate
python clean_orphans.py

```

*You will be prompted before any files are permanently deleted.*

---

## 🧪 Testing

To ensure the service is running correctly, use the provided test suite:

```bash
python test_api.py

```

If successful, it will report:

* ✅ Upload successful
* ✅ VIEW successful
